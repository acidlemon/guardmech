# seacle 生成ファイルは手で編集しない

## 背景 / 罠の説明

`backend/persistence/db/*_gen.go` は先頭に
`// Code generated by seacle.Generator DO NOT EDIT` を持つ生成ファイル。
`Table()` / `Columns()` / `PrimaryKeys()` / `Values()` / `Scan()` 等、
ORM（`github.com/acidlemon/seacle`）が要求するメソッド一式をここで実装している。

生成元は `backend/gen/main.go`。`prepareGenset()` に
「Row 型 → テーブル名 → 出力先ファイル」の対応表が書かれており、
`//go:generate go run github.com/acidlemon/guardmech/backend/gen` から
`cd backend && go generate ./...` で再生成される。

対応する非生成ファイル（例 `principal.go` に対する `principal_gen.go`）の struct 定義には
`db:"column_name,..."` タグが付いており、このタグを Generator が読む。
カラムを増減・改名したいときは、まず非生成ファイルの struct 定義とタグを変更し、
`prepareGenset()` の対応表もあわせて確認してから `go generate ./...` を実行する。

`*Row` 型（例 `PrincipalRow`）は RDB 表現であり、ビジネスロジックのドメインモデルは
`backend/app/logic/membership/` 側にある。両者の変換は
`backend/persistence/query.go`（読み取り）と `backend/persistence/command.go`（書き込み）、
および各 `db/*.go`（非生成側）の `xxxRowFromEntity` / `ToEntity` が担う。
`*Row` とドメインモデルを混同して直接やり取りしない。

## チェックリスト

1. スキーマ変更は `schema.sql` を先に直す
2. 対応する非生成ファイル（`db/xxx.go`）の struct タグを直す
3. 新規テーブル・型を追加した場合は `backend/gen/main.go` の `prepareGenset()` に追記する
4. `cd backend && go generate ./...` を実行し `*_gen.go` を再生成する
5. `*_gen.go` を手で直接編集しない
